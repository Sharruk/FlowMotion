Experiment 5: Explore Git and GitHub commands and Practice Source code
management
Scenario:
Implement version control for your project using Git/GitHub. Apply and analyze
workflows to improve efficiency, collaboration, and maintainability of your project
source code.
Marks Distribution:
• Workflow Analysis – 40
• Demo of Git operations – 40
• Documentation – 20
Tasks :
Repository Setup (K3 | CO4 | PI: 1.3.1, 2.2.4, 3.4.1)
• Apply knowledge of version control to create a Git repository for your project.
• Construct and organize branches for features, development, and main.
• Record and diagram repository structure including branch hierarchy.
Version Control Operations (K3 | CO4 | PI: 3.4.1, 5.1.2)
• Apply Git commands to commit code incrementally, merge branches, and resolve
conflicts.
• Build understanding of relationships between commits, branches, and merges.
• Record commit history, branching relationships, and merge results
systematically.
Workflow Analysis & Documentation (K4 | CO4 | PI: 2.2.4, 3.4.1)
• Analyze and examine the efficiency of the implemented Git workflow.
• Compare alternative branching and merging strategies for effectiveness in
collaborative development.
• Document commands, branch structures, workflow screenshots, and justify
version control strategy for maintainability and project quality.
